div 디스플레이 속성? - 블럭
왜? 브라우저가 그렇게 설정해서.
디스플레이 초기값? inline

inline 특성? 상하단 margin값을 줄수 없다. 사이즈가 알아서 조정된다.
전깃줄을 상상해봐. 뱀처럼 길게 연결된다.
width: 200px
height: 200px 왜 안될까? 인라인이라서. 개념자체가 없음. 일렬이라.

padding은 된다. 상하공간은 차지하지않기 떄문에. 대신 좌우는 안된다. 

width는 상속이 안된다.

inline 엘리먼츠의 베이스라인은 텍스트 기준.
이미지의 특성? : inline

그래서 이미지를 넣을때 옆에 텍스트가 있다면 헝클어진다. 
vertical-align : top을 넣어주면 기준점이 바뀐다.
vertical-align: baseline; 이면 텍스트 기준으로 일렬로 맞춘다.
각 박스의 사이즈가 다르더라도 텍스트 기준으로 정렬을 한다.

line-height : 기본은 노멀.
텍스트가 한줄에 차지하는 높이를 지정한다. 줄 노트에서 각 간격지정으로 생각하면 쉬움.

클래스를 두개이상 넣으면 좋다. 그냥 한칸 띄워주면 됨.

여러 클래스를 지정하고자 함?
.fruit.lemon 붙여서 적자.
.fruit .lemon으로 띄워쓰면? fruit의 자손 lemon이 되어버림..
